# 2026-07-10-021：移除 Vitest 空测试兜底

## 基本信息

- 日期：2026-07-10
- 类型：测试配置 / 质量门禁
- 状态：已完成
- 关联提交：无，当前目录尚未初始化 Git 仓库

## 变动摘要

从主 Vitest 配置中移除 `passWithNoTests: true`。项目已经有真实单元和集成测试，继续允许空测试集通过会掩盖测试匹配配置错误。

## 变动原因

早期工程骨架阶段为了允许无测试文件时通过门禁，曾启用 `passWithNoTests`。当前已有 14 个测试文件和 66 个测试，应恢复 Vitest 默认行为，让测试匹配异常能够失败暴露。

## 影响范围

- 影响模块：Vitest 主测试配置。
- 影响对象：`pnpm test`、`pnpm quality`。
- 行为变化：如果未来主测试配置没有匹配到任何测试，`pnpm test` 将失败。
- 风险等级：低；现有测试集已验证正常匹配并通过。
