# 2026-07-13-014：新增 app.provide app-level 注入 API

## 基本信息

- 日期：2026-07-13
- 类型：功能 / 文档 / 测试
- 状态：已完成
- 关联提交：无，当前目录尚未初始化 Git 仓库

## 变动摘要

为 `createApp()` 返回的 app 实例新增 `provide(key, value)`。每个 app 实例维护独立的 app-level providers map，`mount()` 时传入渲染链路，组件 `inject()` 在组件祖先 providers 之后回退读取 app-level provider。插件可以在 `install` 或函数插件内调用 `app.provide()`。

## 变动原因

Solace 已有组件级 `provide()` / `inject()` 和最小插件 API `app.use()`，但插件缺少向整棵 app 组件树提供依赖值的入口。本次补齐 app-level dependency injection，同时保持组件级 provider 优先级。

## 影响范围

- 影响模块：app runtime、renderer patch 链路、component instance、inject lookup、app/component 单元测试、package consumer smoke、API 文档、README。
- 影响对象：公开 TypeScript 类型 `App`。
- 行为变化：app 实例支持 `provide()`，root 与 descendant 组件可注入 app-level 值；组件 provider 覆盖 app-level provider。
- 风险等级：中；新增 public API，并将 app-level provides 贯穿 renderer/component 主流程。
