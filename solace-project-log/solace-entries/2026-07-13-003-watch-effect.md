## 变动摘要

新增 `watchEffect()` 响应式 API。该 API 会立即运行传入函数，自动追踪函数内读取的响应式依赖，并在依赖变化时重新运行；返回的 stop 函数可停止后续触发。同时将 package `files` 从整个 `docs` 目录收窄为公开 Markdown 文档，避免内部执行计划进入 tarball。

## 变动原因

README 将 `watchEffect()` 列为后续候选 API。它与现有 `ReactiveEffect`、`watch()` 和响应式测试结构天然匹配，是在不扩展组件上下文边界的前提下补齐响应式体验的最小增量。

## 影响范围

- 影响模块：reactivity、package root exports、package manifest、API 文档、README。
- 影响对象：`watchEffect()` 源码入口、ESM/CJS package root 导出、公共 API 说明、发布文件白名单。
- 行为变化：用户可从 `solace` 根入口导入 `watchEffect`。
- 风险等级：低；复用现有 `ReactiveEffect` 停止和依赖清理机制，不改变 `watch()`、`effect()` 的现有行为。
