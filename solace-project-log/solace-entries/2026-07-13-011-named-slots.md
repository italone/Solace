## 变动摘要

为组件 children 新增 named slots 支持。组件现在可以通过 `h(Component, null, { header, default })` 接收多个 slot 函数，并在 setup context 中通过 `slots.header?.()`、`slots.default?.()` 等方式读取。

## 变动原因

Solace 已支持 default slot，但无法表达 header/footer/sidebar 等多区域组件内容。本次补齐 named slots 的最小能力，同时保留普通 children 自动映射到 `slots.default` 的既有行为。

## 影响范围

- 影响模块：VNode children 类型、`h()` overload、组件 slots 初始化、组件单元测试、package consumer smoke、API 文档、README。
- 影响对象：组件 children 传参方式、公开 TypeScript 类型 `Slots`、`VNodeSlots`、`ComponentVNodeChildren`。
- 行为变化：组件 children 可以是 slot 函数字典；slot 更新时会清理旧 slot key。
- 风险等级：中；变更触及 VNode/组件公共层，但不改 renderer diff 主流程。
